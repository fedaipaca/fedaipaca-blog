# console.log: Simple Debugging Patterns

A tiny, practical guide to using `console.log` for quick debugging during development.

## When to use it

- Quick sanity checks (is a value defined?)
- Inspecting objects during development
- Logging execution flow in small scripts

## Examples

Basic usage:

```js
const name = "Fedai";
console.log("Hello", name);
```

Logging objects with context:

```js
const user = { id: 1, name: "Fedai", role: "author" };
console.log("user:", user);
```

Timing a block of code:

```js
console.time("load");
// ... some work
console.timeEnd("load");
```

## Notes

- Remove or gate verbose logs in production builds.
- Consider using structured logging or a proper logger for larger apps.
